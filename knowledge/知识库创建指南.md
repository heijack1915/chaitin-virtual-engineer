# 知识库创建指南

知识库是 Chaitin Virtual Engineer 的核心能力来源。每个产品的安装、卸载、升级、排错指南都以知识库形式加载到程序中，AI 和自动化流程均基于知识库内容执行。

## 知识库格式

知识库是一个 ZIP 压缩包，内部结构如下：

```
safeline-knowledge.zip
├── manifest.json           # 元信息（必填）
├── sources/                # 原始文档（可选）
│   └── install-guide.md
└── wiki/                   # Wiki 内容（必填）
    └── index.md            # 主要操作手册
```

## manifest.json

```json
{
  "name": "SafeLine WAF",
  "version": "25.03.004",
  "description": "雷池(SafeLine) WAF 软件版部署手册"
}
```

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| name | string | 是 | 产品名称，显示在知识库列表中 |
| version | string | 否 | 知识库版本号 |
| description | string | 否 | 简要描述 |

## wiki/index.md

这是 AI 和自动化流程**实际读取的文件**。内容应该包含：

- 产品简介和版本信息
- **部署模式说明**（如果支持多种部署方式）
- **硬件和系统要求**
- **完整的安装步骤**（含命令）
- **卸载步骤**（含命令）
- **升级步骤**（含命令）
- **常见问题和排查方法**

### 编写要点

1. **命令必须是可直接执行的完整命令**，AI 会按照文档中的命令逐步在目标主机上执行
2. **每一步应该有清晰的步骤编号和说明**
3. **标注每个步骤的预期结果**，方便 AI 判断是否成功
4. **使用代码块包裹命令**，方便解析
5. **注明前提条件**（如 root 权限、端口开放等）

### 示例

```markdown
# SafeLine WAF 软件版部署手册

版本：V1.1（25.03.004）
产品：雷池(SafeLine) WAF 软件版

## 硬件要求

- CPU：≥ 4 核，支持 ssse3 和 avx2
- 内存：≥ 8 GB
- 数据盘：≥ 500 GB

## 安装

### 新安装

```bash
# 1. 校验安装包
md5sum safeline-2-software-installer-*.bin

# 2. 创建安装目录
mkdir -p /data/safeline && cd /data/safeline

# 3. 解压安装包（在安装目录内直接解压）
yes "" | ./safeline-2-software-installer-*.bin -p 安装包密码

# 4. 初始化部署
minion setup -p /data/safeline -t Software

# 5. 启动服务
systemctl start minion

# 6. 验证（应全部 up，无 unhealthy）
docker ps -a
```

## 卸载

```bash
systemctl stop minion
docker rm -f $(docker ps -a -q)
rm -fr /data/safeline
rm -rf /etc/minion /etc/systemd/system/minion.service
rm -f /usr/sbin/minion /usr/sbin/minion.*
systemctl daemon-reload
```

## sources/ 目录

存放原始文档（PDF 转换的 Markdown、官方手册等），作为知识库的参考来源。程序不会直接读取此目录，仅供知识库维护者参考。

## 打包

```bash
zip -r safeline-knowledge.zip manifest.json wiki/ sources/
```

## 导入

在程序的「知识库」标签页中点击「导入」，选择打包好的 ZIP 文件即可。导入后 AI 对话和自动化安装流程会自动基于该知识库内容执行。

## 多产品支持

当前程序支持同时加载多个知识库。每个产品的知识库独立打包，分别导入即可。程序未来计划支持以下产品：
